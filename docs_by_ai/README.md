# diary_generator 仕様書（AI作成版）

このディレクトリは、**実装コードを一次情報**として整理した仕様書群です。  

この文書は現行コードの挙動を説明するものであり、仕様の正本ではない。
仕様の変更や判断は docs/ 配下の文書を優先する。

## この仕様書の目的

- 人間が全体像を短時間で把握できること
- AIが「変更要求 -> 影響範囲特定 -> 実装」の流れを取りやすいこと
- 将来、要件変更時の差分管理がしやすいこと

## 読む順序（推奨）

1. `01_system_overview.md`
2. `02_architecture_and_flow.md`
3. `03_data_contracts.md`
4. `04_page_and_url_spec.md`
5. `05_json_output_spec.md`
6. `06_execution_and_operations.md`
7. `07_change_guide_for_ai.md`

## 記述ルール

- 「現状仕様」と「変更時に壊れやすい点」を分けて記載
- 仕様はコード起点で記述（願望仕様は書かない）
- パス・キー名・URL形式は実装どおりに固定表記

## スコープ外

- デザイン意図（CSS詳細の設計思想）
- インフラ詳細（VPS設定値や秘密情報）
- 将来の理想設計（現状コードにない機能）
