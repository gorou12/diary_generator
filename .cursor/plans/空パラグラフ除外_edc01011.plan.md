# 空パラグラフ除外 Plan

## 変更対象

[diary_generator/contents.py](d:/Develop/diary_generator/diary_generator/contents.py) のみ

## 根本原因

`_fetch_diary_page` の `else` ブランチで、すべての非 `heading_3` ブロックに対して無条件に `_update_topic_last_edited_time` を呼んでいる。

```python
# 現状（293行目付近）
else:
    if not current_topic["title"]:
        continue
    _update_topic_last_edited_time(current_topic, last_edited_time)  # 空パラグラフも対象
    ...
    normalized_block = _normalize_block(block)
    if normalized_block:
        current_topic["blocks"].append(normalized_block)
```

トピック間の空パラグラフはNotionが管理する区切りであり、ユーザーが意図的に編集した「コンテンツ」ではない。これらを `last_edited_time` の更新対象およびキャッシュ対象から除外する。

## 変更内容

`_fetch_diary_page` の `else` ブランチに、空 `paragraph` をスキップするガード節を1行追加する:

```python
else:
    if not current_topic["title"]:
        continue
    # 空のparagraphはlast_edited_timeにもblocksにも含めない
    if block_type == "paragraph" and not text_content:
        continue
    _update_topic_last_edited_time(current_topic, last_edited_time)
    ...
```

## 判断の根拠

- `text_content` は `_extract_text_content` が返す `plain_text` の連結をstripしたもの。空文字列 = コンテンツなし。
- `block_type == "paragraph"` に限定することで、画像（`image`）やコードブロック（`code`）など本文コンテンツを持つ可能性がある他の型を巻き込まない。
- `_normalize_block` も空paragraphに対して `{"block_id": "...", "type": "paragraph"}` を返して `blocks` に追加してしまうが、今回の `continue` でその処理にも到達しなくなるため、キャッシュへの混入も同時に解消される。

## 変更しないもの

- `_finalize_topic` — 変更不要
- `_update_topic_last_edited_time` — 変更不要
- `_normalize_block` — 変更不要
- その他すべてのファイル — 変更不要
